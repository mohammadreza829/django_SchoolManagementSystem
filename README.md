### پلتفرم آموزش آنلاین مبتنی بر Django — مدیریت دوره، آزمون هوشمند، چت‌روم و پنل مدیریت

**مکتب‌پلاس** یک سامانهٔ کامل آموزش آنلاین (LMS) فارسی و راست‌به‌چپ است که با **Django** ساخته شده. این پروژه چرخهٔ کامل یک مدرسه/آموزشگاه آنلاین را پوشش می‌دهد: از ثبت‌نام کاربران و انتشار دوره‌ها گرفته تا برگزاری آزمون با تصحیح خودکار، گفتگوی زندهٔ دوره، گزارش‌گیری و پنل مدیریت ویژهٔ اساتید و مدیران.

> [!NOTE]
> این مخزن یک پروژهٔ آموزشی/نمونه‌کار است. تنظیمات پیش‌فرض برای **توسعه (development)** پیکربندی شده‌اند و پیش از استقرار در محیط واقعی باید سخت‌سازی شوند (به بخش [نکات امنیتی](#-نکات-امنیتی-و-آماده‌سازی-برای-production) مراجعه کنید).

## 🧭 دربارهٔ پروژه

مکتب‌پلاس به‌گونه‌ای طراحی شده که چهار نقش اصلی یک سامانهٔ آموزشی را کنار هم بنشاند:

- **دانش‌آموز/دانشجو** دوره‌ها را مرور و خریداری می‌کند، جلسات را تماشا می‌کند، پیشرفتش ثبت می‌شود، در آزمون‌ها شرکت می‌کند و در چت‌روم دوره با استاد و هم‌کلاسی‌ها گفتگو می‌کند.
- **استاد** دوره و آزمون می‌سازد، سوالات بانک سوال را مدیریت می‌کند، نتایج دانش‌آموزان را می‌بیند و در چت‌روم اعلان می‌گذارد.
- **مدیر (Admin)** به همهٔ بخش‌ها و پنل جنگو دسترسی کامل دارد.
- **والدین** به‌عنوان نقش پیش‌بینی‌شده در مدل کاربر وجود دارند (قابل توسعه).

کل رابط کاربری فارسی و **راست‌به‌چپ (RTL)** است و با فونت **Vazirmatn** و **Tailwind CSS** ظاهری مدرن و واکنش‌گرا دارد.

## 🛠 پشتهٔ فناوری (Tech Stack)

| لایه | فناوری‌ها |
|------|-----------|
| **بک‌اند** | Python 3.12+، Django 6.0 (معماری function-based views) |
| **پایگاه‌داده** | SQLite (پیش‌فرض توسعه) — قابل تعویض با PostgreSQL/MySQL |
| **احراز هویت** | مدل کاربر سفارشی (`accounts.User`) با نقش‌ها، مبتنی بر سشن Django |
| **رابط کاربری** | Tailwind CSS (CDN)، Alpine.js، Lucide Icons، Motion.js، Font Awesome (پنل) |
| **فونت** | Vazirmatn (Google Fonts) |
| **فرمول ریاضی** | KaTeX — رندر فرمول‌های LaTeX در سوالات و گزینه‌های آزمون |
| **نمودارها** | Chart.js — داشبورد پیشرفت و آمار پنل مدیریت |
| **رسانه/تصاویر** | Pillow برای پردازش `ImageField` |
| **چت** | HTTP polling (بدون نیاز به WebSocket/Channels) |

## ✨ قابلیت‌ها و ویژگی‌های کامل

در ادامه فهرست کامل قابلیت‌های سایت به تفکیک ماژول آمده است.

### 👤 حساب کاربری و پروفایل (اپ `accounts`)
- ثبت‌نام عمومی دانش‌آموز با اعتبارسنجی **کد ملی** و **شمارهٔ موبایل ایران** (الگوی `09xxxxxxxxx`).
- ورود/خروج مبتنی بر سشن با پیام‌های فارسی و پشتیبانی از پارامتر `next`.
- **مدل کاربر سفارشی** با چهار نقش: مدیر، استاد، دانش‌آموز، والدین.
- **پروفایل عمومی**: بیوگرافی، آواتار، تصویر کاور، جنسیت، تاریخ تولد، وب‌سایت، محل سکونت و شبکه‌های اجتماعی (توییتر/اینستاگرام/لینکدین/گیت‌هاب).
- پروفایل‌های اختصاصی **استاد** (تخصص، مدرک) و **دانش‌آموز** (شمارهٔ دانش‌آموزی، سال ورود) که هنگام ساخت کاربر به‌صورت خودکار با Signal ساخته می‌شوند.
- صفحهٔ پروفایل عمومی هر کاربر با نمایش دوره‌های در حال تدریس/ثبت‌نام‌شده.
- ویرایش پروفایل، تغییر رمز عبور، و محاسبهٔ خودکار سن/تولد.
- **سامانهٔ اعلان (Notifications)**: زنگولهٔ بالای صفحه با شمارندهٔ خوانده‌نشده‌ها (به‌صورت ارقام فارسی، و `۹۹+` برای تعداد زیاد).
- **داشبورد شخصی**: تعداد دوره‌ها، ساعت‌های تماشاشده، میانگین پیشرفت، فعالیت‌های اخیر و دوره‌های پیشنهادی.
- لیست کاربران با جستجو (ویژهٔ کارکنان/staff).

### 📚 دوره‌ها و جلسات (اپ `courses`)
- **دسته‌بندی درختی** دوره‌ها (دسته و زیردسته) با شمارش خودکار تعداد دوره در هر دسته.
- لیست دوره‌ها با **فیلتر بر اساس دسته و سطح** (مقدماتی/متوسط/پیشرفته) و **جستجو** روی عنوان، توضیح کوتاه و نام استاد.
- صفحهٔ جزئیات دوره با شمارندهٔ بازدید (مقاوم در برابر race condition با `F()`)، امتیازها، دوره‌های مرتبط و **جلسات پیش‌نمایش رایگان**.
- چند استاد برای هر دوره (رابطهٔ Many-to-Many).
- **انواع محتوای جلسه**: ویدیو (آپلودی یا لینک)، متن آموزشی، و فایل.
- مسیریابی **جلسهٔ قبلی/بعدی**، ثبت **پیشرفت تماشا** و علامت‌گذاری «تکمیل‌شده».
- **فایل‌های ضمیمه** برای هر جلسه با دانلود کنترل‌شده و شمارش دانلود.
- **نظر و پاسخ به نظر** برای جلسات + **لایک** جلسه.
- **امتیازدهی ۱ تا ۵ ستاره** به دوره + بازمحاسبهٔ خودکار میانگین امتیاز با Signal.
- قیمت‌گذاری به تومان، **درصد تخفیف** و محاسبهٔ قیمت نهایی، و تشخیص دورهٔ رایگان.
- مدیریت **ظرفیت دوره** و **مهلت ثبت‌نام**؛ کنترل ظرفیت به‌صورت اتمیک (`select_for_update`) تا از تکمیل بیش از ظرفیت جلوگیری شود.
- صفحهٔ «دوره‌های من» با درصد پیشرفت هر دوره.
- تولید خودکار **slug** (با پشتیبانی یونیکد/فارسی).

### 📝 ثبت‌نام (اپ `Enrollment`)
- مدل واسط `Enrollment` بین دانشجو و دوره با **وضعیت** (فعال/تکمیل‌شده/لغوشده) و **وضعیت پرداخت** (رایگان/در انتظار/پرداخت‌شده/ناموفق).
- نگه‌داری مبلغ پرداختی و درصد پیشرفت، و متد `mark_completed` برای صدور گواهی (آماده برای توسعه).
- به‌روزرسانی خودکار `enroll_count` و پرچم «تکمیل ظرفیت» با Signal.

### 🧪 آزمون و بانک سوال (اپ `quiz`)
- **بانک سوال قابل بازاستفاده**: یک سوال در چند آزمون.
- **موضوعات درختی** برای دسته‌بندی سوالات.
- **پنج نوع سوال**: تک‌گزینه‌ای، چندگزینه‌ای، درست/غلط، پاسخ عددی (با خطای مجاز)، پاسخ کوتاه متنی.
- سطح سختی، بارم (نمره)، تصویر سوال، و **راه‌حل/توضیح** که بعد از آزمون نمایش داده می‌شود.
- پشتیبانی از **فرمول‌های LaTeX** در متن سوال‌ها و گزینه‌ها (رندر با KaTeX).
- تنظیمات آزمون: **محدودیت زمانی**، **حد نصاب قبولی**، **حداکثر دفعات مجاز**، **ترتیب تصادفی سوالات** و نمایش/عدم‌نمایش راه‌حل.
- **تصحیح خودکار** همهٔ انواع سوال + محاسبهٔ نمره، درصد و وضعیت قبولی.
- صفحهٔ نتیجه با نمایش پاسخ صحیح و راه‌حل، و صفحهٔ **«پیشرفت من»** با نمودار Chart.js.
- **کنترل دسترسی**: آزمون‌های متصل به دوره فقط برای ثبت‌نام‌شده‌ها؛ آزمون‌های عمومی برای همه؛ اساتید و مدیران بدون محدودیت.

### 🛎 پنل مدیریت اختصاصی (اپ `panel`)
- **داشبورد آماری** ویژهٔ استاد/مدیر با KPIها و نمودارها (Chart.js): روند شرکت در آزمون طی ۱۴ روز اخیر، میانگین نمره به‌تفکیک آزمون، نسبت قبول/مردود و آخرین تلاش‌ها.
- مدیریت کامل دوره‌ها: **ایجاد/ویرایش/حذف** و **انتشار/پیش‌نویس** با یک کلیک.
- مدیریت کامل آزمون‌ها: ساخت آزمون، **افزودن سوال همراه گزینه‌ها** (با Formset)، حذف سوال و تغییر وضعیت انتشار.
- **گزارش نتایج**: مشاهدهٔ نتایج هر آزمون، آمار قبول/مردود/میانگین و جزئیات تک‌تک تلاش‌ها.
- **خروجی CSV** نتایج آزمون (با BOM یونی‌کد تا اکسل فارسی را درست بخواند).
- دسترسی محدودشده: اساتید فقط دوره‌ها/آزمون‌های خودشان را می‌بینند؛ مدیر همه را.

### 💬 چت‌روم دوره (اپ `chat`)
- هر دوره یک **چت‌روم** اختصاصی دارد.
- پیام‌رسانی بلادرنگ با **polling هر ۴ ثانیه** (بدون WebSocket) از طریق اندپوینت JSON.
- **اعلان استاد**: استاد می‌تواند پیام را به‌عنوان «اعلان» ارسال کند و برای همهٔ دانش‌آموزان ثبت‌نام‌شده **Notification** ساخته می‌شود.
- کنترل دسترسی: فقط استاد دوره، مدیر یا دانش‌آموز ثبت‌نام‌شدهٔ فعال.

### 🧩 قابلیت‌های فرامرزی (Cross-cutting)
- **پنل ادمین جنگو** برای همهٔ مدل‌ها (کاربر، پروفایل، دوره، جلسه، امتیاز، ضمیمه، ثبت‌نام، آزمون، سوال، تلاش، پیام چت و اعلان).
- **Signalها** برای ساخت خودکار پروفایل، تولید slug، بازمحاسبهٔ امتیاز و شمارش ثبت‌نام.
- **Context Processorها** برای در دسترس بودن سراسری دسته‌بندی‌ها و شمارندهٔ اعلان‌ها در همهٔ قالب‌ها.
- **کانورتر URL سفارشی** برای پشتیبانی از slug فارسی در آدرس آزمون‌ها.
- اسکریپت **`seed_demo.py`** برای پر کردن سریع دیتابیس با داده‌های نمونهٔ واقع‌گرایانه.

## 🗃 مدل دادهٔ اصلی

نمودار ساده‌شدهٔ روابط اصلی:

```text
User (نقش: admin/teacher/student/parent)
 ├─< Profile / TeacherProfile / StudentProfile        (پروفایل‌ها)
 ├─< Notification                                     (اعلان‌ها)
 │
Category ──< Course >── (M2M) ── User(teacher)         (اساتید دوره)
                │
                ├─< Lesson ──< LessonProgress / LessonComment / LessonLike / LessonAttachment
                ├─< CourseRating
                ├─< Enrollment >── User(student)        (ثبت‌نام دانشجو)
                ├─< CourseMessage                       (چت‌روم)
                └─< Quiz
                        │
Topic ──< Question ──< Choice                           (بانک سوال)
   Quiz ──(QuizQuestion)──< Question
   Quiz ──< QuizAttempt ──< AttemptAnswer
```

## 🚀 نصب و راه‌اندازی

> این مراحل روی Python 3.12 و Django 6.0 تست شده‌اند.

```bash
# ۱) کلون کردن مخزن
git clone https://github.com/mohammadreza829/django_SchoolManagementSystem.git
cd django_SchoolManagementSystem

# ۲) ساخت و فعال‌سازی محیط مجازی
python3.12 -m venv venv
source venv/bin/activate        # ویندوز: venv\Scripts\activate

# ۳) نصب وابستگی‌ها
pip install --upgrade pip
pip install "Django>=6.0,<6.1" Pillow

# ۴) اعمال مهاجرت‌ها (ساخت جداول دیتابیس)
python manage.py migrate

# ۵) ساخت کاربر مدیر (اختیاری — اگر از داده‌های نمونه استفاده نمی‌کنید)
python manage.py createsuperuser

# ۶) اجرای سرور توسعه
python manage.py runserver
```

سپس در مرورگر به آدرس **http://127.0.0.1:8000/** بروید. پنل ادمین جنگو در **/admin/** و پنل مدیریت اساتید در **/panel/** در دسترس است.

> [!TIP]
> برای راحتی می‌توانید یک فایل `requirements.txt` با محتوای زیر بسازید و با `pip install -r requirements.txt` نصب کنید:
> ```text
> Django>=6.0,<6.1
> Pillow
> ```

## 🌱 داده‌های نمونه و حساب‌های دمو

برای پر کردن دیتابیس با داده‌های واقع‌گرایانه (دوره، جلسه، آزمون، ثبت‌نام، چت و ...) اسکریپت `seed_demo.py` را اجرا کنید:

```bash
python seed_demo.py            # ساخت داده‌های نمونه (idempotent — تکراری نمی‌سازد)
python seed_demo.py --fresh    # ابتدا داده‌های نمونه را پاک و سپس از نو می‌سازد
```

پس از اجرا، با این حساب‌ها می‌توانید وارد شوید (رمز عبور همه: **`demo12345`**):

| نقش | نام کاربری | رمز عبور |
|-----|-----------|----------|
| مدیر | `admin` | `demo12345` |
| استاد | `teacher01` | `demo12345` |
| دانش‌آموز | `student001` | `demo12345` |

> داده‌های نمونه شامل ده‌ها دوره، صدها جلسه و ثبت‌نام، بانک سوال، آزمون با تلاش‌های تصحیح‌شده، پیام‌های چت و اعلان است تا نمودارها و صفحات پر و قابل ارزیابی باشند.

## 🔒 نکات امنیتی و آماده‌سازی برای production

> [!WARNING]
> تنظیمات فعلی برای **توسعه** است. پیش از استقرار حتماً موارد زیر را اصلاح کنید:

- **`SECRET_KEY`** را از کد خارج کرده و از متغیر محیطی بخوانید (کلید فعلی در مخزن قرار دارد و ناامن است).
- **`DEBUG = False`** و مقداردهی صحیح **`ALLOWED_HOSTS`**.
- جایگزینی **SQLite** با یک پایگاه‌دادهٔ تولیدی مانند **PostgreSQL**.
- سرو فایل‌های **media/static** توسط وب‌سرور (Nginx) یا فضای ابری به‌جای سرور توسعهٔ Django.
- استفاده از یک **WSGI/ASGI server** تولیدی (Gunicorn/Uvicorn) پشت وب‌سرور.
- افزودن تنظیمات امنیتی HTTPS (HSTS، Secure Cookies، CSRF trusted origins).
